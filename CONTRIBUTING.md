# Contributing to Termux-Mastery

## Welcome

Thank you for your interest in contributing to Termux-Mastery! Even though this is a solo-maintained repository, community improvements, corrections, and suggestions are warmly welcomed. Your help ensures that this guide remains accurate, clear, and perfectly suited for beginners learning Termux.

## What Contributions Are Welcome

We are always looking for improvements that make learning easier for newcomers. You are highly encouraged to contribute by:

- Fixing typos or grammar errors
- Improving confusing explanations
- Adding missing beginner-friendly examples
- Reporting outdated commands or broken instructions
- Suggesting new beginner topics

## What Is NOT Welcome

To keep this project focused, safe, and aligned with its core goals, the following contributions will **not** be accepted:

- Hacking, exploitation, or attack-oriented content
- Videos or external media links
- Content written in languages other than English
- Advanced topics that go beyond the beginner-to-intermediate scope
- Unsolicited restructuring of the repository or its folders

## How to Contribute

If you would like to make a change, please follow this simple step-by-step process:

1. **Fork** this repository to your own GitHub account.
2. **Edit** the files you want to improve in your forked version.
3. **Commit** your changes with a clear description of what you fixed or added.
4. **Submit a Pull Request (PR)** from your fork back to this main repository.

## Pull Request Rules

To ensure your pull request is reviewed and merged smoothly, please verify that it meets these strict rules:

- Submit only **one topic or fix per pull request**.
- Must follow the existing README template and folder structures.
- Must use clean, standard GitHub Markdown.
- Must be written in **English only**.
- Must maintain a friendly, beginner-accessible tone.

## Reporting Issues

If you find a problem but do not want to fix it yourself, please report it using GitHub Issues. When creating an issue, make sure to:

- Describe the problem clearly.
- Explicitly mention the **folder number and name** where the issue occurs (for example, `03 - Package Management`).

## Review Process and Final Decisions

Please keep in mind that Termux-Mastery is a solo project. The repository owner personally reviews all contributions and retains the final decision on what gets merged. If a contribution does not fit the vision or scope of the project, it may be politely declined. 

## Thank You

Thank you again for taking the time to read these guidelines and for helping make Termux-Mastery an excellent, reliable resource for beginners everywhere. Your support is deeply appreciated!
